Chosen Implementation

How to implement Chosen jQuery plugin.

Chosen is a jQuery plugin that makes long, unwieldy select boxes much more user-friendly.

Example

Implementation

Include jQuery, Chosen plugin and chosen Stylesheet

Include a stylesheet that applies GOV UK design, must come after the chosen.css

Implement Chosen on the required selectors (in this case class name enableChosen). This should be included somewhere in a javascript file, or in script tags. 


   $(document).ready(function(){
       $(".enableChosen").chosen({
          inherit_select_classes: true,
          placeholder_text_multiple: 'Select one or more countries',
          width: 'none' // this breaks it on purpose so css takes over for responsive design
       });
    });

Notes

Touch Devices

Touch screen devices have native methods for handling multi-select, which will be familiar to the user. Chosen also has problems generally on touch devices. Due to the lack of any "hover" paradigm, touch devices often implement a hover on touch, and hence many front end methods are complex to implement. These two factors led us to disable chosen for touch devices using the following change to chosen.js (iPod and iPhone are disabled by default)

Accessibility

ARIA implementation for accessibility is not fully mature, and through using an extra stylesheet to implemnet gov.uk design patterns, we hope to keep the chosen plugin up to date as it matures further.

We are not yet aware of any specific issues with assistive technologies.

Scope

As multi-selects were the target element (as they typically require holding control to select multiple on desktop), only multi-selects have had the design pattern applied. Other select types could also be similarly styled.
